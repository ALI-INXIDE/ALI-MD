### 🎯 Core Features
- **Multi-Device Support** — Works with WhatsApp's latest multi-device technology

### (B) Render Deploy
<a href="https://render.com/deploy" target="_blank">
  <img src="https://img.shields.io/badge/-Render%20Deploy-000000?style=for-the-badge&logo=render&logoColor=white" alt="Render Deploy" width="150" height="28"/>
</a>

<details>
<summary>Deployment Guide</summary>

**Steps:**
1. Fork this repo, then go to [render.com](https://render.com) and sign in.
2. Click **New → Blueprint** and connect your forked repo.
3. Render reads `render.yaml` and **auto-provisions a free PostgreSQL** database linked to your bot.
4. Fill in `SESSION_ID` when prompted. All other vars have defaults.
5. Click **Apply** — done.

> **Note:** Render's free PostgreSQL lasts 90 days. After that, use a free external DB from [neon.tech](https://neon.tech) and paste the URL as `DATABASE_URL`. If left blank the bot falls back to SQLite automatically.

</details>

---

### (C) Railway Deploy
<a href="https://railway.com/new" target="_blank">
  <img src="https://img.shields.io/badge/-Deploy%20on%20Railway-00C2FF?style=for-the-badge&logo=railway&logoColor=white" alt="Railway Deploy" width="170" height="28"/>
</a>

<details>
<summary>Deployment Guide</summary>

**Steps:**
1. Fork this repo and go to [railway.app](https://railway.app).
2. Click **New Project → Deploy from GitHub repo** and select your fork.
3. In your project dashboard, click **+ New → Database → PostgreSQL**.
4. Railway auto-links `DATABASE_URL` to your service — no copy-paste needed.
5. Go to your service **Variables** tab and add:
   - `SESSION_ID` → your `ALI-MD~` session ID
   - `MODE` → `public`
   - `TIME_ZONE` → e.g. `Asia/Karachi`
6. Railway detects the `Dockerfile` and `railway.toml` automatically and deploys.

</details>

### (D) Koyeb Deploy
<a href="https://app.koyeb.com/" target="_blank">
  <img src="https://img.shields.io/badge/-Deploy%20on%20Koyeb-EC4899?style=for-the-badge&logo=koyeb&logoColor=white" alt="Koyeb Deploy" width="170" height="28"/>
</a>

<details>
<summary>Deployment Guide</summary>

**Steps:**
1. Fork this repo and sign in to [koyeb.com](https://koyeb.com).
2. Click **Deploy Now** above (or Create App → GitHub → select your fork).
3. Koyeb has no built-in database — get a **free PostgreSQL** from one of:
   - [neon.tech](https://neon.tech) ← recommended
   - [supabase.com](https://supabase.com)
4. Paste the connection URL as `DATABASE_URL` in your Koyeb service environment variables.
5. Set `SESSION_ID`, `MODE`, `TIME_ZONE` in the same env vars section.
6. Koyeb uses the `Dockerfile` and `koyeb.yaml` config automatically.

> **Tip:** If you skip `DATABASE_URL` the bot will use SQLite on the local disk — data resets on each redeploy. Use a remote DB for persistence.

</details>

## 🔑 Environment Variables

| Variable | Required | Description |
|---|---|---|
| `SESSION_ID` | ✅ Yes | Your WhatsApp pairing session, generated from the pairing site above |
| `DATABASE_URL` | ❌ No | PostgreSQL connection string. Falls back to SQLite if left empty |
| `MODE` | ❌ No | `public`, `private`, or `groups` — default `public` |
| `TIME_ZONE` | ❌ No | e.g. `Asia/Karachi` — used for scheduled features |
| `PREFIX` | ❌ No | Bot command prefix — default `.` |

> All owner-configurable settings (bot name, footer, chatbot mode, anti-features, etc.) can also be changed live via in-chat commands — no redeploy required.

## ❓ FAQ

<details>
<summary><b>My session keeps disconnecting, what do I do?</b></summary>
<br>
Generate a fresh SESSION_ID from the pairing site and make sure only one instance of the bot is running with that session at a time.
</details>

<details>
<summary><b>Can I use SQLite instead of PostgreSQL?</b></summary>
<br>
Yes — just leave <code>DATABASE_URL</code> empty and the bot automatically uses local SQLite storage.
</details>

<details>
<summary><b>Will my WhatsApp number get banned?</b></summary>
<br>
Using unofficial WhatsApp bots always carries some risk. Avoid mass-messaging or spam-like behavior to minimize the chance of a ban. See the disclaimer below.
</details>
